1. 常见问题


问: 什么是Double Commander？

答: Double Commander 是一个跨平台的双面板文件管理器。详细信息请见 这里。



问：这个版本有什么新功能？

答: 变更列表可以在 维基页面 上找到。但该项目目前正在开发中，因此后续的 夜间构建 包含许多新功能和改进。



问：这是 Linux 的 Total Commander 吗？

答: 不，这是一款来自其他开发者的程序。它现在支持 Windows、GNU/Linux 操作系统，使用 GTK 和 Qt 工具包（依赖于 Lazarus，因此目前支持 GTK2 和 Qt5）以及 macOS，同时 Double Commander 也可以在 FreeBSD Ports 集合中找到。



问：这个程序和我最喜欢的文件管理器（Total Commander）有什么区别？

答: 该项目的目标是创建一个在操作上接近 TC 的程序，并通过插件和配置文件实现最大兼容性。然而，Double Commander 具有一些重要的优势：它是免费的，可以在不同的操作系统上运行，并且是开源的。



问：我如何联系 Double Commander 的开发者？

答: 项目的 官方论坛 或 GitHub。

确保您使用的是程序的最新版本，并且在报告错误或请求新功能之前，检查开发（alpha）版本是个好主意。 另见 #117。

（回到顶部）

2. 与 GNU/Linux 相关的问题


问：我应该下载哪个版本？

答: 如果你使用 KDE、LXQt 或 Fly，请下载 Qt 版本；在其他情况下（Cinnamon、GNOME、LXDE、MATE 或 Xfce），请下载 GTK 工具包的版本。但这不是严格的规则。



问：项目仓库在哪里？

答: 一些 GNU/Linux 发行版的仓库：

  • 发布：GTK2 或 Qt5

  • alpha ("nightly") 构建: GTK2 或 Qt5

还提供便携版本（详细信息请见这里）。



问: Total Commander 的插件无法工作？！

答: 它们可以工作，但并不是所有的。为 Windows 编译的 TC 插件只能与 Windows 版本的 Double Commander 一起使用。 但是如果插件有开源代码，你可以尝试为 GNU/Linux 构建它们（或者也可以请插件开发者来做 :））。

有许多适用于 Linux 的插件和跨平台插件（请参见Wiki、库plugins、doublecmd-plugins等） 以及官方论坛上的几个插件讨论话题（例如，话题1，话题2）。



问：我可以像在 Nautilus 或 Thunar 中一样访问网络资源吗？

答: 是的，DC 可以使用 GVfs。 GVfs支持多种协议：NFS、SMB、FTP、SFTP、WebDAV、MTP、手机（Windows、Android、Apple）、媒体播放器等。

首先，应该安装软件包 gvfs 、 gvfs-backends 和 gvfs-fuse 。请注意，一些发行版没有软件包 gvfs-backends 和 gvfs-fuse ，因为它们是软件包 gvfs 的一部分，或者它们可以是多个名为 gvfs-backends-* 或 gvfs-* 的软件包。

现在在地址栏中插入 GVfs 路径：右键单击地址栏以进行编辑（或调用内部命令 cm_EditPath ）。

例如，访问权限

  • FTP: ftp://ftpuser@ftpserver
  • SMB: smb://servername/sharename/
  • WebDAV: davs://servername

好吧，访问 FTP（也适用于 SSH+SCP）和 SMB 时，您可以使用默认的 WFX 插件 FTP 和 Samba。



问：点击托盘图标会显示一个包含“恢复”和“退出”选项的菜单，而不是恢复窗口（仅限 GTK2）。

答: 添加一个通用系统托盘支持库：例如，在 Debian/Ubuntu 中打包 libappindicator1 或在 Arch Linux 中打包 libappindicator-gtk2 。

（回到顶部）

3. 接口配置


问：我如何更改面板的颜色设置？

答: 要更改颜色，您必须转到配置 > 选项... > 颜色 > 文件面板。这些是两个面板的全局设置，可以被覆盖，请参见配置 > 选项... > 文件视图 > 列 > 自定义列。



问：按钮 R 和 All 在颜色设置中是什么？

答: 它们如下：

  • R  – 将颜色恢复为默认。

  • All  – 将所选颜色应用于所有列。



问：某一列中的文本覆盖了另一列的文本！

答: 配置 > 选项... > 文件视图 > 列，勾选“按列宽裁剪文本”。



问：如何将我自己的图标与某些文件类型关联？

答: 你应该执行以下操作：配置 > 选项... > 文件关联。



问：如何在按钮面板上创建分隔符？

答: 从类别 分隔符 添加一个按钮。



问：DC 还有其他图标主题吗？

答: 现在我们只知道一个小主题 DoubleMac，但安装说明已经过时（见下文）。
您可以自己制作图标主题，这很简单（我们将使用“ MyDCTheme”作为示例）。

如何制作：
- 打开 DC 文件夹，查看 "pixmaps/dctheme" 的目录结构；
- 复制 "dctheme" 并命名为 "mydctheme";
- 打开 "mydctheme/index.theme" 并更改主题名称：将 "Name=DCTheme" 替换为 "Name=MyDCTheme"；
- 现在更换图标。


注意：
1. 不要使用默认的图标替换！您的图标将在 DC 更新后被替换。
2.如果您想保留一些现有的默认图标，只需在“mydctheme”中删除它们，DC 将从默认主题中使用它们。
3. 一些非标准图标尺寸（例如 40x40、60x60 等）用于 HiDPI 显示器。
4. 另见 <CustomIcons>。


如何安装：

便携版
将您的“mydctheme”文件夹复制（或移动）到靠近 doublecmd.exe 的“pixmaps”文件夹中，重启 DC，进入主菜单中的“配置”> 选项... > 图标 > 图标主题，选择您的主题，然后再次重启 DC。

已安装版本

前往主菜单中的“配置” > 选项... > 配置 > 目录 > 图标主题：在这里您将看到可能的目录列表。

不要使用 /usr/share/doublecmd/pixmaps、C:\Program Files\Double Commander\pixmaps 或其他系统目录：这需要根/admin 权限，并且这会干扰 Linux 中软件包管理器的工作或 Windows 中的 DC 安装程序。 DC 通常建议以下额外路径（除了靠近doublecmd.exe 的 "pixmaps" 文件夹）：

  • Windows XP: C:\Documents and Settings\<UserName>\Local Settings\doublecmd\pixmaps
  • Windows Vista/7+: C:\Users\<UserName>\AppData\Local\doublecmd\pixmaps
  • 类 Unix 操作系统：/home/<UserName>/.local/share/doublecmd/pixmaps

如果目录不存在，请创建它。

现在重启 DC，进入主菜单中的“配置” > 选项... > 图标 > 图标主题，选择你的主题，然后再次重启 DC。



问：如何使功能键和地址栏变大？

答: 尺寸取决于字体大小，有三种方式：

简单快捷：您可以使用 Ctrl +鼠标滚轮，它适用于以下界面元素或 DC 的一部分：
- 左侧和右侧面板中的文件列表；
- 当前目录（地址）栏；
- 功能键按钮栏；
- 树视图菜单；
- 查找文件对话框中的搜索结果；
- 内部编辑器；
- 内部查看器（如果查看器显示文本，则此操作将更改字体大小；如果显示图像，则该操作将作为放大/缩小命令工作）。

2. 在主菜单中转到“配置” > 选项... > 字体。
注意：所有可能的功能仅在版本 1.0.0 及以上可用，对于 0.9.x 及以下版本，请参见第三项。

3. 关闭 DC > 打开 doublecmd.xml  > 找到标签 <Fonts></Fonts> 并手动更改界面元素的 <Size>。



问：文件面板：如何改变元素（字符串）的高度及其之间的间隔？

答: 视图取决于以下参数：

  • 配置 > 选项... > 字体 > 主字体
  • 配置 > 选项... > 图标 > 图标大小 > 文件面板
  • <ExtraLineSpan> （来自 <FilesViews>）

4. 与控制和导航相关的问题


问：在使用 Space 键选择后，是否可以自动将光标向下移动一行？

答: 你应该执行以下操作：配置 > 选项... > 文件视图 > 当使用空格键选择文件时，移动到下一个文件（与插入相同）。


问：是否有能力创建文件操作队列？

答: 在 0.4.6 alpha 版本中实现了文件操作队列，并已添加到下一个稳定版本中。


问： Ctrl+A 不工作，如何启用它？

答: 你应该执行以下操作：配置 > 选项... > 键 > 热键 > 在“主”类别中，滚动查看命令 cm_MarkMarkAll 并为其设置 Ctrl+A 快捷键。


问：当目录中的所有文件被选中时，无法使用鼠标取消选择它们。

答: 几种方法：

  • 在文件列表的开头使用“..”项（但如果文件列表很长，则需要滚动查看）。
  • 使用主菜单：标记 > 取消全选。
  • 在工具栏上添加“取消全选”按钮（内部命令 cm_MarkUnmarkAll ）。
  • 转到主菜单中的“配置” > 选项... > 鼠标 > 鼠标选择 > 启用“通过点击图标”：第一次点击将取消标记一个文件，现在点击他的名字。（如何在缩略图视图中使用，请参见此参数的 描述。）


问：如何在这个程序中创建我自己的上下文菜单？

答: 将按钮添加到“菜单”类别的工具栏中。请参见本节以获取示例。


问：如何自定义上下文菜单，例如，添加“使用其他程序打开”选项？

答: 这可以被纳入Double Commander；然而，向上下文菜单的“操作”子菜单中添加命令很简单。以下是如何以根权限打开任何目录的示例：

  1. 在文件类型下，点击添加，然后在弹出窗口中输入，例如 Dir。
  2. 在扩展中，点击添加，然后在弹出窗口中输入，例如 folder。
  3. 在“操作”下，点击“添加”，然后在下面的“操作：编辑”行中，输入所需的操作，例如 以root用户身份打开
  4. 在命令编辑行中，输入所需的命令：例如 pkexec doublecmd %p
  5. 之后，只需右键单击任何文件夹，选择“以root用户身份打开”，然后输入root用户的密码。 :)


问：如何自动解压并运行打开的压缩文件中的文件？每次都在压缩文件的属性窗口中使用“解压并执行”按钮很不方便。

答: 查看<AutoExtractOpenMask>的描述。


问：当文件从一个目录复制/移动到另一个目录时，评论（通过 descript.ion 文件）可以被复制/移动吗？

答: 是的，这是可能的：前往配置 > 选项 > 文件操作，并启用“与文件/文件夹一起处理评论”。

5. 与内置查看器 ( F3 ) 和编辑器 ( F4 ) 相关的问题


问：当启用 Compiz 时，编辑器和浏览器窗口会随机出现位置。

答: 编辑 Compiz 设置：放置窗口 > 固定位置的窗口和窗口规则 > 固定大小的窗口。

(class=Doublecmd) & (title=/)


问：字体看起来很奇怪（横向字符相互覆盖等）或选择功能不正常。

答: 你应该在查看器和编辑器中使用等宽字体。

注意：“Monospace”是一个字体别名，值可能并不来自等宽字体系列。如果您遇到问题，请尝试在Double Commander设置中指定一个真实的等宽字体。


问：查看器在某些图像上无法工作，但在大多数情况下可以正常工作。

答: 用于查看图像的组件不支持某些 JPEG 文件。


问：在某些文本文件中，行超出了窗口的边缘。

答: 在查看器中启用“换行”选项（在“视图”菜单中）。内部编辑器不支持换行。


问：你能添加列编辑功能吗（像在 Notepad++、UltraEdit 等中那样）？

答: 内部编辑器支持列和行选择模式以及多光标，查看键盘快捷键 在这里。


问：我如何使用正则表达式在 UTF-8 中进行搜索？

答: 此功能已添加到版本 1.0.0。需要支持 8 位代码单元和启用 Unicode 支持的 PCRE2 库（通常默认启用）：

  • Linux: 需要 libpcre2-8.so.0 。例如，在 Debian/Ubuntu 中的包 libpcre2-8-0 或在 Arch Linux 中的 pcre2 。

  • Windows: 需要 libpcre2-8.dll （默认情况下，Double Commander 发行版包含此 DLL）。

  • macOS：需要 libpcre2-8.dylib 。


问：如何在内部编辑器中更改语法高亮方案或添加新的方案？

答: DC 使用两个组件：

1. SynEdit 具有内置语法高亮显示器。如果您想更改任何高亮显示器，您应该更改 DC 或 Lazarus 源代码。但您可以轻松更改使用的颜色和文件扩展名列表：请前往主菜单中的“配置” > 选项... > 工具 > 编辑器 > 高亮显示器。不要忘记为每种文件类型保存自定义设置。

2. SynUniHighlighter 用于自定义语法高亮方案 (.hgl)，您可以尝试自己查找或创建这些文件（见下文）。

这些文件通常是基于 XML 的文件，您可以在文本编辑器中打开并阅读/更改。文件的开头是<General>标签（在<Info>内部）：

1) 旧格式：<General> 具有标签
- <Name>: 语法高亮菜单的名称;
- <FileTypeName>: 支持的文件扩展名列表。

2) 新格式：<General> 具有属性
- 名称: 语法高亮 菜单的名称;
- 扩展名: 支持的文件扩展名列表。

（DC 支持这两种格式。）

如何创建或编辑：

使用 UniHighlighter 编辑器或 HglEditor (下载)。这两个程序是为 Windows 编写的，但您可以使用 Wine。HglEditor 将以新格式保存 HGL 文件。
还有各种语法高亮文件可供选择。
在工具 > 编辑器 > 文本高亮部分，您可以更改文本和背景颜色以及字体样式。


如何添加：

1) 便携版
将您的 HGL 文件复制（或移动）到靠近doublecmd.exe 的“highlighters”文件夹中，然后重启 DC。

2) 安装版
前往主菜单中的“配置” > 选项... > 配置 > 目录 > 高亮：，在这里您将看到可能的目录列表。
不要使用/usr/share/doublecmd/highlighters、C:\Program Files\Double Commander\highlighters 或其他系统目录：这需要root/admin 权限，并且这会干扰 Linux 中软件包管理器的工作或 Windows 中的 DC 安装程序。 DC 通常建议以下额外路径（除了doublecmd.exe 附近的“highlighters”文件夹）：

  • Windows XP: C:\Documents and Settings\<UserName>\Local Settings\doublecmd\highlighters
  • Windows Vista/7+: C:\Users\<UserName>\AppData\Local\doublecmd\highlighters
  • 类 Unix 操作系统: /home/<UserName>/.local/share/doublecmd/highlighters

如果目录不存在，请创建它。现在重启 DC。

附加功能：

1) 其他 子菜单

额外的 语法高亮 菜单项：如果菜单太长，则可以将一些项目移动到 其他 子菜单。 只需向Other添加新属性和值 1 到<General>标签，然后重启 DC。
例如，在……之前
  <General Name="AutoIt v3*" Extensions="AU3"/>
和之后
  <General Name="AutoIt v3*" Extensions="AU3" Other="1"/>

6. 与文件面板相关的问题


问：我在列表中没有找到适合文件面板的日期格式！我该如何添加自己的格式？

答: 日期和时间格式易于配置：前往配置 > 选项 > 文件视图 > 格式化 > 日期和时间格式，并使用 此表。


问：我想查看“名称”列中的文件名及其扩展名，可以吗？

答: 是的，这是可能的。请在主菜单中转到“配置” > 选项... > 文件视图 > 列 > 自定义列，现在创建一个新的列集或更改现有的默认集：默认情况下，DC 使用 GETFILENAMENOEXT 字段作为名称，请改用 GETFILENAME。


问：可以通过点击列标题（“日期”、“大小”等）来对文件夹进行排序吗？

答: 是的，您可以启用此功能：在主菜单中转到“配置” > 选项... > 文件视图 > 排序 > 排序目录，并使用“像文件一样排序并优先显示”或“像文件一样排序”。


问：如何查看符号链接的目标对象？

7. 插件

问: “这不是一个有效的插件！”是什么意思？

答: 这通常意味着依赖项存在问题：并非所有必需的库在您的系统中都可用。如果插件的描述中没有详细信息，无法联系作者或使用搜索引擎找到解决方案，那么请尝试以下方法：

  • Windows：FileInfo 或 PEViewer 插件，实用工具如 DLL Export Viewer 或 Dependency Walker。
  • Linux：使用终端和命令 ldd （最简单的方法， ldd pluginname | grep "not found" ），GNU Binutils 或 AnyELF 插件。

如果这是一个用 Lua 编写的 WDX 插件：

1. Lua 库不可用：请参见 所需的 DLL。

2. 该脚本需要一个额外的模块：请查看脚本的描述或联系作者。

3. 脚本中的错误。为了调试，您可以在终端或高级代码编辑器/IDE（例如，ZeroBrane Studio）中使用 Lua。
如果您使用Double Commander功能：创建一个带有内部命令cm_ExecuteScript的按钮，并使用可用的功能（DC.LogWrite，Dialogs.MessageBox，Clipbrd.SetAsText或将结果保存到文件中）。


问：ftp.wfx：我尝试使用 FTPS（FTP+SSL），但出现“522 SSL 连接失败：需要会话重用”。

答: 该服务器需要会话重用支持，FTP 插件支持此功能，但需要一个支持 SSL 和 TLS 协议的库（请参见插件的 描述）。将库放在 doublecmd.exe 附近或在 PATH 中（Windows），或安装适当的包（Linux 和其他类 Unix 系统）。


问：ftp.wfx：与某些服务器不兼容！连接后，我收到错误：“104，连接被对方重置”和“CSOCK 错误 连接被对方重置”或“10060，连接超时”和“CSOCK 错误 连接超时”。

答: 需要一个支持 SSL 和 TLS 协议的库（请参见插件的 描述）。将库放在 doublecmd.exe 附近或在 PATH 中（Windows），或安装适当的包（Linux 和其他类 Unix 系统）。
