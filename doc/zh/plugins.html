1. 打包插件 (WCX)


Base64

Base64 插件提供了根据 Base64 方案对文件进行编码和解码的能力，这是使用可打印的 ASCII 字符编码二进制数据的标准之一。


CPIO

用于解压缩 CPIO 归档的打包插件。


DEB

用于解压 dpkg 包管理系统（Debian 及其衍生版）包文件的 Packer 插件。

Linux 和其他类 Unix 操作系统（在 Windows 中，Double Commander 使用 7Zip 插件来支持此格式）。


RPM

用于解包 Red Hat 软件包管理系统（Red Hat Linux、SUSE Linux 等）包文件的 Packer 插件。


7Zip

用于处理 .7z、.arj、.cab、.cpio、.cramfs、.deb、.dmg、.fat、.hfs、.iso、.lha、.lzh、.ntfs、.squashfs、.taz、.vhd、.wim、.xar 和 .z 文件的 Packer 插件。我们还可以添加其他 7-Zip 支持的文件类型。

注意：某些格式为只读，意味着仅可进行内容查看、解压和测试。

仅限 Windows。


解压缩 RAR 文件

解压 RAR 档案的打包插件。

要使用此插件，我们需要UnRAR库，插件将使用 unrar.dll （Windows）， libunrar.dylib （macOS）， libunrar.so 或 libunrar.so.5 （Linux 及其他类 Unix 操作系统）。

Windows 版的 Double Commander 和 Linux 的便携版本包含必要的库，在其他情况下，我们可以使用操作系统中获取库的常用方法（例如，在 Debian/Ubuntu 中使用包 libunrar5 或在 Arch Linux 中使用 libunrar ）。否则，我们可以尝试在这个页面上找到合适的变体（文件必须复制到系统目录或 Double Commander 可执行文件所在的目录）。

该插件还支持将文件打包成 RAR 压缩文件，为此，我们需要在插件设置中指定 WinRAR.exe 的路径（仅限 Windows）或 rar 命令行工具。


Zip

用于处理 .bz2、.gz、.jar、.lzma、.tar、.tbz、.tgz、.tlz、.txz、.tzst、.xz、.zip、.zipx 和 .zst 压缩档案的 Packer 插件。该插件还可以打开基于 ZIP 的文件（如 EPUB、Office Open XML、OpenDocument 格式、CRX、XPI 等）。

需要额外的压缩库：

  • bzip2: bz2.dll （Windows）， libbz2.dylib （macOS）， libbz2.so.1 （Linux 和其他类 Unix 操作系统）。

  • XZ: liblzma.dll （Windows）， liblzma.dylib （macOS）， liblzma.so.5 （Linux 和其他类 Unix 操作系统）。

  • Zstandard: libzstd.dll （Windows）， libzstd.dylib （macOS）， libzstd.so.1 （Linux 和其他类 Unix 操作系统）。

Double Commander for Windows 的发行版包含所有必要的库，这些库通常在 Linux 的桌面/工作站中默认安装。

2. 内容插件 (WDX)


内置 Exif 插件

该程序内置了 Exif 内容插件，可以获取一些可以存储在 JPEG 文件元数据中的数据：日期、像素尺寸、图像方向和相机信息。

仅支持 JPEG 图像。


音频信息

该内容插件显示有关音频文件的信息，包括技术信息（时长、比特率、通道数等）和元数据（标题、艺术家等）。

支持以下类型的元数据：ID3v1（1.0，1.1），ID3v2（2.2，2.3，2.4），APE（1.0，2.0），Vorbis 注释。


DEB

此内容插件显示有关 dpkg 包管理系统（Debian 及其基于的发行版）的包文件信息：包名称、版本、描述、依赖关系等。


RPM

此内容插件显示有关红帽软件包管理系统（红帽 Linux、SUSE Linux 等）的软件包文件的信息：软件包名称、版本、描述等。


Lua 脚本

plugins/wdx/scripts 目录包含几个用 Lua 编写的内容插件示例（有关详细信息，请参见 Lua 脚本 帮助页面）。

这些插件没有在程序设置中添加。

  • descriptionwdx.lua  – 从 descript.ion 读取文件描述。

  • fulltextodtwdx.lua – 在 OpenDocument 文本文件（ODT）中查找文本。需要 odt2txt 。

  • simplewdx.lua  – 带有多个字段的简单示例。

  • textlinewdx.lua – 逐行返回纯文本文件（.txt, .ini）的前五行，从第一行到第五行。

3. 文件系统插件 (WFX)


FTP协议

这是一个文件系统插件，用于处理以下网络协议：FTP、FTPS、FTPES、SFTP 和 SSH+SCP。该插件还支持与代理服务器（HTTP、SOCKS 4 或 SOCKS 5）一起使用。

需要额外的库：

1) SSL 和 TLS 协议

  • OpenSSL 加密库 (3.1, 3.0, 1.1.1, 1.1.0, 1.0.2, 1.0.1)：FTP 插件将使用 ssleay32.dll 和 libeay32.dll （Windows）， libssl.dylib 和 libcrypto.dylib （macOS）或 libssl.so 和 libcrypto.so （Linux 和其他类 Unix 操作系统），其中 "N" 是版本号。库名称通常包含版本号，插件包含常见名称的列表，所需文件将自动在系统中找到。
    编译好的 Windows 版 OpenSSL 二进制文件可以在 这里 找到，在其他情况下，我们可以使用操作系统的包管理器（例如，在 Debian/Ubuntu 中使用包 libssl3 或 libssl1.1 ，在 Arch Linux 中使用 openssl ，在 macOS（Homebrew）中使用 openssl ）。

  • 或者，在 Linux 中，插件可以使用GnuTLS库（ libgnutls.so.N ，其中“N”是版本号）。

2) SSH 协议

libssh 库：FTP 插件将使用 libssh2.dll （Windows）、 libssh2.dylib （macOS）或 libssh2.so.1 （Linux 和其他类 Unix 操作系统）。

Double Commander for Windows 的发行版包含此库，在其他情况下，我们可以使用操作系统中的常规方法获取库（例如，在 Debian/Ubuntu 中的包 libssh2-1 ，在 Arch Linux 中的 libssh2 ，在 macOS（Homebrew）中的 libssh2 ）。

插件允许使用 SSH 认证代理（例如，来自 OpenSSH 工具套件的 ssh-agent，来自 PuTTY 的 Pageant）。


Samba协议

这是一个用于处理网络目录（SMB/CIFS）的文件系统插件。

仅限 Linux。

4. Lister 插件 (WLX)


Mac预览

通用（办公文档格式、图像、视频和音频文件等）插件使用 快速查看，如同在 Finder 中一样。

仅限 macOS。


资源管理器预览

用于显示丰富预览的插件。IPreviewHandler 接口支持多种格式（取决于操作系统的版本和安装的应用程序），默认情况下，仅将 HTML 文件添加到检测字符串中。

注意：预览处理程序可能无法显示文件中包含的所有信息。

仅限 Windows。


富文本格式

查看 RTF 文件的插件。

仅限 Windows。


WlxMplayer

查看视频文件的插件。需要 mplayer 。

仅限 Linux。


Windows 媒体播放器

播放音频和视频文件的插件。此插件使用 IWMPPlayer4 接口（ActiveX），需要 Windows Media Player 9 或更高版本。

仅限 Windows。

5. 搜索插件 (DSX)


DSX 定位

此插件使用 locate 及其文件数据库进行搜索。我们可以通过文件名、部分文件名或通过通配符进行搜索（符号“*”表示匹配任意数量的字符）。

Linux 和其他类 Unix 操作系统。
