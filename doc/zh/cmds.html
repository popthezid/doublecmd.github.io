1. 命令类别表

内部命令可以按类别分组，以下是它们：

指令类别表
类别描述
活动面板
 这些命令适用于当前选定的面板，左侧或右侧。
左侧面板
 这些命令适用于左侧面板，无论当前选择的是哪个。
右侧面板
 这些命令适用于右侧面板，无论当前选择的是哪个。
文件操作
 这些命令适用于当前选定面板中的当前选定项。
它包括查看/复制/移动/重命名/删除/打包/解包/分割等操作。
这些是可以对您的 文件产生直接影响 的命令。
配置
 Double Commander的访问选项和配置。
网络
 与计算机网络直接相关。
杂项
 不可避免的类别，包含那些我们不知道该将其归入哪个类别的命令！
已尽力避免将过多命令归入此类别。
标记
 选择将进行文件操作的项目。
剪贴板
 系统剪贴板与选定项目之间的交互。
导航
 在不同文件夹之间移动，以根据其位置访问不同的项目。
帮助
 访问Double Commander的集成帮助文件。
窗口
 与Double Commander窗口及其外观和行为相关的命令，类似于其他计算机应用程序。
命令行
 与命令行交互以查看过去的命令并帮助调用新的命令。
工具
 对于需要超过一次点击的操作。这些操作通常会显示另一个窗口，以便让Double Commander执行一些更复杂的任务。
查看
 与数据文件、系统文件等的可见性和外观相关的命令。
标签
 通过创建新标签与标签进行交互，从一个目录导航到另一个目录等。
日志
 与 DC 日志文件相关的操作。

2.4. 文件操作

这些命令适用于当前选定面板中的当前选定项。
它包括查看/复制/移动/重命名/删除/打包/解包/分割等操作。
这些可能是对您的文件有直接影响的命令。

文件操作类别
名称描述
cm_View
F3
 在查看器程序中打开文件。

该命令支持一个参数，允许强制查看模式（在这种情况下，将忽略插件，但您可以稍后在查看器窗口中切换模式）。 请查看此页面上模式的描述。

参数 值 描述
mode text 显示为纯文本
bin 显示为二进制
hex 显示为十六进制
dec 显示为十进制

注意：如果没有选择文件或仅选择了一个文件，则此参数将有效。
cm_Edit
F4
 在编辑器中打开文件（内部或外部，请参见工具 > 编辑器）。
cm_EditNew
Shift+F4
 在当前目录中创建一个新文本文件并在编辑器中打开（请参见 工具 > 编辑器）或打开现有文件（如果已存在同名文件）。
您可以输入文件的完整名称（包括路径），它将被创建在指定的目录中。
cm_Copy
F5
 从源复制项目到目标。

参数 值 描述
confirmation 1/true/on/yes 显示确认对话框（默认）
0/false/off/no 不显示确认对话框
queueid <queue identifier> 默认情况下，将复制操作放入特定队列，仅在未显示确认对话框时有效，值为 0 表示将是独立操作（不排队）

示例：“cm_Copy confirmation=1”将提示用户确认复制，无论文件操作的复制操作确认设置如何。
cm_CopyNoAsk
 从源位置复制项目到目标位置，无需确认。
cm_CopySamePanel
Shift+F5
 在同一目录中复制项目。
cm_Rename
F6
 重命名或移动项目。

参数 值 描述
confirmation 1/true/on/yes 显示确认对话框（默认）
0/false/off/no 不显示确认对话框
queueid <queue identifier> 默认情况下，将复制操作放入特定队列，仅在未显示确认对话框时有效，值为 0 表示将是独立操作（不排队）

示例：“cm_Rename confirmation=1”将提示用户确认重命名/移动，无论文件操作的移动操作确认设置如何。
cm_RenameNoAsk
 重命名或移动项目时无需确认。
cm_RenameOnly
F2，Shift+F6
 在同一目录中重命名文件。
cm_MakeDir
F7
 创建一个新目录。
cm_Delete
F8, Del
 将选定的项目删除到回收站或永久删除。
删除到回收站可能并不适用于所有平台。
删除到回收站或永久删除时会显示不同的确认消息。

参数 值 描述
confirmation 1/true/on/yes 显示确认对话框（默认）
0/false/off/no 不显示确认对话框（请小心！）
trashcan 1/true/on/yes 删除到回收站
0/false/off/no 永久删除
setting （默认）根据文件操作中的全局设置进行相应的操作，如果该设置已启用，则会删除到回收站，否则将永久删除
reversesetting 与设定相反的方式行事

Alt+Del
 擦除 文件。
cm_PackFiles
Alt+F5
 打包 项目到档案中。

参数 值 描述

 如果选择了一个目录，将打开该目录以显示其内容。
如果选择了一个文件，忽略其扩展名，将根据检测到的内容尝试将其作为归档打开（请参见其他帮助页面中的 ID 配置）。

Alt+F9
 解压 一个或多个选定的档案。

2.5. 配置

Double Commander的访问选项和配置。

类别 "配置"
名称描述
cm_Options
 打开配置对话框。
命令支持一个参数以跳转到特定的配置部分。
这里是一个链接，用于可能的参数。
cm_ConfigToolbars
 访问工具栏的配置。
cm_ConfigDirHotList
Ctrl+Shift+D
 访问常用文件夹列表的配置。
它相当于 cm_WorkWithDirectoryHotlist 命令，带有以下参数：
action=config source=%Ds
cm_WorkWithDirectoryHotlist
 访问“常用文件夹列表”配置窗口。
通过这些参数，您可以控制将对热名单执行哪些操作。

参数 值 描述
action add 添加条目（目标将根据配置中的选项添加）
addsrconly 将仅添加一个源条目
addboth 将添加一个条目和目标
config 配置一个条目
show 简单地显示配置窗口
addsel 添加当前选定的目录
（其他参数被忽略）
directconfig 配置指定的第“n”条目
(nothing) 同样是“添加”
source source_path 将目录添加/配置为“源”
(nothing) 活动面板所在的目录
target target_path 将目录添加/配置为“目标”
(nothing) 目录中不活动面板所在的位置
index 0-999 如果操作=directconfig，则从热列表中获取第“n”条目
(nothing) 索引 0 将被使用

示例：“cm_WorkWithDirectoryHotlist action=addboth”
这将把当前活动和非活动框架的目录分别作为源和目标添加到目录热列表中。
cm_FileAssoc
 打开文件关联 配置。
cm_ConfigFolderTabs
 访问与文件夹标签相关的设置配置。
cm_ConfigFavoriteTabs
 访问收藏标签的配置。
cm_ConfigTreeViewMenus
 访问树视图菜单的配置。这是我们可以配置默认使用树视图菜单选择样式的命令的地方。
cm_ConfigTreeViewMenusColors
 访问与树视图菜单部分界面相关的颜色配置。
cm_ConfigSearches
 访问文件搜索的一般选项配置。
cm_ConfigHotKeys
 访问热键的配置。
cm_ConfigSavePos
 将保存 DC 主窗口的当前状态和位置。
cm_ConfigSaveSettings
 将保存所有当前设置和历史记录，就像您关闭应用程序并恢复一样。
这可能会很有用，如果您在长时间会话中使用了 DC，并且您想保存当前上下文，以防系统崩溃，并且您不想在重新启动 DC 时丢失当前历史记录。
cm_ConfigArchivers
 将启动 外部压缩程序 的配置页面。
cm_ConfigTooltips
 将为工具提示启动配置页面。
cm_ConfigPlugins
 访问插件的配置。
cm_AddPlugin
 此命令使添加插件变得简单，支持两种方式：
选择一个插件文件，Double Commander将自动检测其类型（WCX、WDX、WFX、WLX 或 DSX），打开所需的设置部分并将其添加到列表中。
选择包含插件和特殊文件 pluginst.inf 的归档文件（插件文件和 pluginst.inf 必须位于归档文件的根目录中）： 归档的内容将被提取到适当的目录，插件将被添加到设置中，Double Commander将打开相应的设置部分。如果设置存储在用户配置文件中，Double Commander将使用用户目录中的“plugins”文件夹：
  • Windows XP: C:\Documents and Settings\<UserName>\Local Settings\doublecmd\plugins
  • Windows Vista/7+: C:\Users\<UserName>\AppData\Local\doublecmd\plugins
  • Unix-like 操作系统: /home/<UserName>/.local/share/doublecmd/plugins
在便携版中，Double Commander将使用程序目录中的“插件”文件夹。

注意：第一种添加插件的方法需要先将文件放置在合适的位置，但是在这种情况下，如果插件无法加载（例如，如果操作系统不包含必要的库），Double Commander将报告错误。

文件名可以作为参数传递。

需要超过点击的操作。
这些操作通常会显示另一个窗口，以便Double Commander执行一些更复杂的任务。

类别 "工具"
名称描述

cm_Search

 打开搜索对话框，使用默认参数或从“默认搜索模板”选项中使用保存的模板（请参见文件操作 > 文件搜索）。

 如果我们已经启动了几个带有 cm_AddNewSearch 的搜索实例，这个命令将允许我们一次性关闭并从内存中移除它们。

如果使用外部差异程序，则可以使用“dir”参数（不带引号）：在这种情况下，左侧和右侧文件面板的当前目录将传递给该程序。
cm_DebugShowCommandParameters
 将简单地显示一条消息，内容为作为参数传递的信息。
这可能有助于验证您传递给命令的参数。
cm_DoAnyCmCommand
Shift+F12
 将显示一个内部命令选择窗口，用户可以选择任何可能的内部命令并执行它。
